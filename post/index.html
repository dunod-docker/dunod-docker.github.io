Dockerfile HEALTHCHECK

2 septembre 2016 Thomas Garlot

3 minutes de lecture

Une autre nouveauté de Docker v1.12 est l’ajout de l’instruction HEALTHCHECK pour le Dockerfile. En effet, avec le nouveau mode Swarm, un docker ps n’est plus suffisant: si le conteneur “tourne” mais que l’application à l’intérieur du conteneur ne renvoie que des erreurs, comment Docker Swarm peut-il savoir qu’il doit redémarrer le conteneur? C’est là que la l’instruction HEALTHCHECK entre en jeux. Avant Docker 1.12, il fallait utiliser des mécanismes tel que les health checks de Consul ou ceux de Mesosphere.
